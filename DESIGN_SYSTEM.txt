# InAmigos Foundation Website Design System

> This document defines every visual and UI decision for the project.
> 
> 
> The developer should follow this specification exactly unless a technical limitation makes it impossible.
> 
> No visual decisions should be guessed.
> 

---

# Design Philosophy

The website should feel calm, modern, trustworthy and premium.

It should communicate hope and professionalism through simplicity rather than excessive decoration.

The overall experience should resemble a modern nonprofit organization rather than a college assignment.

Visual inspiration comes from

https://hartregie.nl/

Only use the following as inspiration:

- color mood
- typography style
- spacing
- visual hierarchy
- clean layouts

Do NOT copy:

- layouts
- graphics
- illustrations
- icons
- branding
- components

The final website must be original.

---

# CSS Guidelines

Use CSS Variables

Organize styles into logical sections

Reuse utility classes

Avoid duplicate styling

Keep selectors readable

Avoid !important unless absolutely necessary.

---

# Performance

Optimize images

Lazy-load gallery images

Minimize unnecessary CSS

Keep animations lightweight

Avoid unnecessary JavaScript.

# Final Design Goal

The finished website should feel like the official landing page of a modern nonprofit organization.

The interface should immediately communicate

- Trust
- Compassion
- Professionalism
- Transparency
- Community

A visitor should feel inspired to learn more and participate without the design distracting from the organization's mission.
